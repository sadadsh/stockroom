# Stockroom M7 — Full Projects (audit, health, Editor, BOM, procurement, buildability, fab-prep) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan slice-by-slice. Steps use checkbox (`- [ ]`) syntax for tracking. Each slice (M7a…M7i) is a vertical, independently-shippable surface built TDD, verified on Linux, adversarially reviewed on its committed diff, and Windows-pixel-verified before it is called done — exactly as M6a…M6k were.

**Goal:** Bring the full Projects capability of the retired PyQt app into Stockroom at zero regression — register/open KiCad projects, audit + health them against the library, edit every project-wide setting (design rules, net classes, board setup, presets, project meta, object conform), run ERC/DRC, build a correctly-grouped priced BOM with procurement + exports, fuse it all into one Buildability verdict, and prepare a fab package — with every KiCad write routed through the byte-preserving Layer 0.

**Architecture:** Reuse-by-extraction. The retired app's Qt-free `nd_*` Projects modules and its ~28 pure BOM/procurement functions (today buried in the PyQt `LibraryManager`) are lifted into Qt-free `stockroom/projects/*` modules. Per the governing invariant (spec §2.1) the **clean-room owns every WRITE**: reused logic is READ/COMPUTE only, and every KiCad file edit re-routes through Stockroom's Layer-0 span-splice editor (`sexp/`) inside a git-atomic `Transaction`. Projects are a brand-new first-class concept in the engine (there is none today): a JSON `ProjectRecord` source of truth plus a derived, rebuildable `ProjectIndex`, mirroring `PartRecord` + `LibraryIndex`.

**Tech Stack:** Python 3.12 backend (FastAPI, the existing `sexp`/`mutation`/`vcs`/`store`/`model` layers, `kicad-cli`), React + TypeScript + Tailwind + TanStack Query frontend (vitest/RTL/jsdom), permissive third-party lifts: KinJector schema + `merge_dicts` (MIT), KiBoM grouping rules (MIT), KiField derived-view pattern, KiKit for panelization (invoked, not vendored).

## Global Constraints

Every task's requirements implicitly include this section. Values are copied verbatim from the spec + repo `CLAUDE.md`.

- **Zero Qt in the backend.** `stockroom.api` and every `stockroom/*` module import zero PyQt (`PyQt5`, `QtCore`, `QtWidgets`, `QtGui`, `QSettings`). Reuse is reuse-by-extraction into Qt-free modules whose behavior is verified against the old tests. A CI gate greps the backend package and fails on any hit. **Never import `fp_render` or `LibraryManager`** (both drag PyQt5 at import).
- **Every write goes through Layer 0.** The byte-preserving s-expression layer (`sexp/SexpDocument`) is the only thing that edits `.kicad_pcb`/`.kicad_sch`; a new targeted `.kicad_pro` JSON editor is the only thing that edits `.kicad_pro`; the Layer-1 `Transaction` is the only committer. Edits are scoped token/key substitution, never a re-serialize, never `pcbnew.Save`, never the retired `nd_*` `.bak`+writer path. The writer losslessly passes through tokens it does not model and fails loud on overlapping/corrupt edits.
- **Fail-proof / atomic.** Every mutation is one git-backed atomic `Transaction`: a single scoped commit, or restore every touched path and leave zero trace. Add `.kicad_pro` to the transaction's JSON-validated suffix set so a malformed project file aborts + rolls back like everything else. **Commit-time asset gate**: a Projects write fails if a record references an asset (symbol/footprint/3D/datasheet) not committed in the same tree. Honest degradation — no swallowed errors; partial results labeled partial.
- **Design contract** (`docs/design/*.md`): no em dashes anywhere; Title Case for interactive labels (buttons, headings, tags, menu items); sentence case for body prose (real sentences, not fragments); 8px card radius / 6px control radius invariant; tokens only (spacing via the design tokens, never scattered literals); category hues ≥3:1 contrast in both themes.
- **Honest completion.** Never claim "0 regressions", "fully verified", "flawless", or "done" off tests alone. State exactly what was exercised and where. Windows CI is the release gate; Linux/offscreen green is necessary, never sufficient. File I/O passes `encoding="utf-8"`; never `str(Path)` for display (use `.as_posix()`).
- **Backend seam is TDD, RED-first, BEFORE its frontend.** Full-stack slices build the pytest seam RED→GREEN first, then the frontend lands on it — exactly as M6d/e/f/i/k did.

## Architecture decisions (locked)

1. **Projects are registered by external path; the ProjectRecord lives in-repo, the KiCad files stay external.** A KiCad PCB project is a separate thing that *consumes* the library (the old app operated on the external NETDECK repo). So `ProjectRecord` (id, humanized name, absolute project dir, discovered `.kicad_pro`/`.kicad_pcb`/`.kicad_sch` sheet paths, cached audit/ERC/DRC digest, git root) is a **registration + cache** stored in the library repo at `libraries/<profile>/projects/<id>.json`; the actual project files remain at their external location. Project-file **writes route through a `Transaction` bound to the project's OWN git repo** (`GitRepo(project_git_root)`); if the project dir is not a git repo, the write is refused with an honest "initialize git for this project first" (offer a one-click `git init`) — this is what makes the commit-time asset gate meaningful. *(Open decision OD1 — default = this.)*
2. **The clean-room owns every write; reuse is COMPUTE only.** The nd_* write *mechanisms* are NOT called. Their COMPUTE is extracted (the psm rule-id sets + `.kicad_pro` key schema, the KinJector definitions+assignments net-class model, the fab/design preset math, the object-conform diff of what-to-set). The actual byte edit is performed by `sexp/SexpDocument` (`.kicad_pcb`/`.kicad_sch`) and the new `kicad/project_settings.py` (`.kicad_pro`) inside a `Transaction`. `nd_board_setup`'s selector logic is ported onto `SexpDocument` as a strong reference, not invoked. *(Open decision OD3 — default = port compute → Layer 0.)*
3. **Projects data model mirrors the library exactly.** `model/project.py` `ProjectRecord` (canonical JSON, sorted keys / 2-space / trailing newline for minimal diffs) is the source of truth; `store/project_store.py` `ProjectStore` (register/list/open/delete, each a scoped commit) mirrors `ProfileStore`; `store/project_index.py` `ProjectIndex` is a derived, rebuildable, never-committed SQLite table built FROM the JSON, rebuilt on load + after pull, exactly like `LibraryIndex`.
4. **Per-project ERC/DRC + audit cache is one source of truth.** A digest cached on the `ProjectRecord` (or in `AppContext`) is read by Overview, Health, and the Buildability verdict, and invalidated on any Prepare/Restore/write — the gen [7] pattern from the retired app, so the three surfaces can never disagree.
5. **KiBoM grouping is added on top of the app's MPN-primary grouping, not swapped in.** Value-normalization + alias-equivalence + DNF-regex are lifted (MIT) and layered onto the extracted `_bom_from_components`, preserving MPN-primary + manufacturer-in-key + value-as-MPN promotion.
6. **The Prepare/Complete-All fill flow (M7f) requires a Qt-free `library_core` first.** `nd_library_fill` is the one hard-Qt module; its ~25 `LM` symbol/footprint text helpers (`qualify_*`, `set_symbol_*`, parsers) are re-homed into `stockroom/library_core.py` (Qt-free, verified against the old tests) before fill is reimplemented against it and Stockroom's link ops. *(Open decision OD5 — default = in scope for M7f.)*
7. **Preset/standard stores are repo-committed portable JSON** (fab presets, design-rule schemes, size templates, net-class vault standards) so they resolve identically on any machine (Req 13), stored at `libraries/<profile>/standards/*.json` and edited through a `Transaction`. *(Open decision OD2 — default = repo-committed.)*
8. **Honest degradation everywhere.** kicad-cli absent → the ERC/DRC/fab surfaces show an honest "KiCad CLI not found" state (Stockroom already tolerates cli-absence), never a fabricated pass. A project whose files moved → an honest "files not found, re-register" state, never a crash.
